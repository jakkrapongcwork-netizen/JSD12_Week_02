### CRUD - create, read, update, delete						
		Read Operations					
		เช่น					
			find() and findOne()				
				db.collection.find(<query>, <projection>, <options>)
    ## query
        - ใช้เป็นเงื่อนไขในการเลือก
        - ทำหน้าที่กำหนดเงื่อนไขที่ Documents จะต้องมีคุณสมบัติครบถ้วนตามที่ระบุไว้ จึงจะถูกเลือกนำมาแสดงผลในชุดผลลัพธ์
        - เขียนอยู่ในรูปแบบของ Object: { <field>: <value>, ... }
    ## projection
        - ใช้สำหรับกำหนดว่าฟิลด์ (Fields) ใดบ้างที่ควรจะแสดงผล หรือไม่ต้องการ
        - สร้าง Document ใหม่
        - รูปแบบไวยากรณ์ (Syntax) และการตั้งค่า
            - เขียนในรูปแบบ Object: { <field>: <include|exclude>, ... }
                - Include (แสดงผล): ใช้ค่า 1 หรือ true เพื่อระบุว่าต้องการให้ฟิลด์นั้นปรากฏในผลลัพธ์
                - Exclude (ซ่อนผล): ใช้ค่า 0 หรือ false เพื่อระบุว่าไม่ต้องการให้ฟิลด์นั้นปรากฏในผลลัพธ์
    ## options
        - ใช้สำหรับกำหนดเงื่อนไขเพิ่มเติมในการแสดงผลข้อมูลที่ดึงออกมาจากฐานข้อมูล
        - ช่วยควบคุมลักษณะการทำงานที่นอกเหนือจากการกรองข้อมูล เช่น การเรียงลำดับ การจำกัดจำนวน หรือการข้ามลำดับข้อมูล
        - รูปแบบไวยากรณ์ (Syntax)
            - เขียนอยู่ในรูปแบบ Object: { <option1>: <value1>, <option2>: <value2>, ... }



							
### Characteristics of a MongoDB Document:						
		คุณลักษณะสำคัญของ Document ใน MongoDB ซึ่งเป็นหน่วยพื้นฐานในการจัดเก็บข้อมูล โดยมีรายละเอียดดังนี้ครับ:					
			1. Key-Value Pairs (คู่ของคีย์และค่า): 				
				ข้อมูลใน Document จะถูกจัดเก็บในรูปแบบคู่ของ Key (ซึ่งเป็น String) และ Value (ซึ่งมีได้หลายประเภท เช่น String, Integer, Array หรือแม้แต่ Document อื่นๆ).			
			2. Dynamic Schema (สคีมาแบบไดนามิก): 				
				MongoDB มีโครงสร้างที่ยืดหยุ่น Document ที่อยู่ใน Collection เดียวกันไม่จำเป็นต้องมีชุดฟิลด์ที่เหมือนกัน และฟิลด์เดียวกันในแต่ละ Document ก็สามารถมีประเภทข้อมูลที่แตกต่างกันได้.			
			3. Nested Documents (เอกสารแบบซ้อน): 				
				Document สามารถเก็บ Document อื่นซ้อนไว้ภายในได้ (Sub-documents) ซึ่งช่วยให้สามารถสร้างโครงสร้างข้อมูลที่มีความซับซ้อนได้ในชุดเดียว.			
			4. BSON Format (รูปแบบ BSON): 				
				ในการทำงานภายใน MongoDB จะจัดเก็บ Document ในรูปแบบ BSON (Binary JSON) ซึ่งเป็นการขยายโมเดลมาจาก JSON เพื่อให้รองรับประเภทข้อมูลที่เพิ่มขึ้น และมีประสิทธิภาพสูงในการจัดเก็บและสแกนข้อมูล			
							
### Aggregation Pipeline

        - การปฏิบัติการที่สามารถนำมาประกอบกันได้ ซึ่งเรียกว่า stages (สเตจ) จะถูกนำมาจัดเรียงตามลำดับที่คุณสร้างขึ้น
        - เอกสารข้อมูล (Documents) จะไหลผ่านสเตจเหล่านี้ ซึ่งส่งผลให้ได้ชุดข้อมูลที่ถูกแปลงสภาพแล้ว
        - ใช้สำหรับการค้นหา (Search), การรวมกลุ่มข้อมูล (Aggregate) และการประมวลผลชุดข้อมูลที่มีความซับซ้อนในปริมาณมาก
        - Aggregation pipeline เปรียบเสมือน สายการผลิต (production line) และแต่ละ stage ก็คือแต่ละขั้นตอนการทำงานภายในสายการผลิตนั้น
        - ผลลัพธ์ที่ได้ก็คือ ชุดเอกสารข้อมูลสุดท้าย (final set of documents)

    ## Common-sense ordering for performance (the “rule of thumb”)    
        ลำดับที่ช่วยให้ระบบทำงานได้เร็วขึ้นโดยทั่วไปคือ: 
            1. Filter (กรอง)
                เช่น $match
                    - $match ทำหน้าที่กรอง Documents ในคอลเลกชัน
            2. Project (เลือกฟิลด์)
                - เลือกเฉพาะข้อมูลที่ใช้
                เช่น $project
            3. Unwind (คลายอาร์เรย์)
                - $unwind ใช้สำหรับแยกเอกสาร (Document) ชุดเดียวที่มีฟิลด์เป็นอาร์เรย์ ให้กลายเป็นเอกสารหลาย ๆ ชุดตามจำนวนสมาชิกในอาร์เรย์นั้น
            4. Group (จัดกลุ่ม)
                - $group จะนำเอกสารที่มี "คีย์" หรือ "ค่าในฟิลด์" ที่เหมือนกันมาจัดรวมไว้ในกลุ่ม
                - เมื่อจัดกลุ่มแล้ว เราสามารถคำนวณหาค่าสรุปผลของข้อมูลในแต่ละกลุ่มได้ โดยใช้ตัวดำเนินการสะสม (Accumulators) เช่น การนับจำนวน (count), การหาผลรวม (sum), หรือการหาค่าเฉลี่ย (average)
            5. Sort (เรียงลำดับ)
            6. Limit (จำกัดจำนวน)
                - อาจจะมีการเรียงลำดับก่อนที่จะทำ Limit
            7. Join (เชื่อมตาราง)
                เช่น $lookup
                    - ทำหน้าที่ในการเชื่อมโยงข้อมูลระหว่างคอลเลกชัน
                    มาผนวกเข้ากับเอกสารในคอลเลกชันหลักที่เรากำลังประมวลผลอยู่
                    - ใช้เพื่อรวมข้อมูลที่ถูกเก็บแยกกันไว้คนละคอลเลกชัน (ในฐานข้อมูลเดียวกัน) เพื่อนำมาประมวลผลหรือแสดงผลร่วมกันในชุดเดียว
