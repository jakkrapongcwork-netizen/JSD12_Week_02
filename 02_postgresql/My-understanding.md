### JOIN Concept			
	1. INNER JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK
	2. LEFT JOIN : 	มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่ง PK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null
	3. RIGHT JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่ง FK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null
	4. FULL OUTER JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่งทั้งฝั่ง PK และ FK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null

### SQL Normalization					
	1. 1NF				
		ห้าม list หรือ มีหลายรายการใน Column เดียว			
	2. 2NF				
		แยกข้อมูลที่ไม่ขึ้นกับ PK ทั้งหมด			
		เช่น			
			PK = (order_id + product_id)		
			แต่เราเก็บ product_name		
			❌ product_name		
			เพราะมันขึ้นกับ product_id อย่างเดียว		
			👉 ต้องแยก table Product		
	3. 3NF				
		ห้าม column พึ่ง column			
		Column ที่ขึ้นกับ column อื่น (ที่ไม่ใช่ PK)			
		เช่น			
			| customer_id | zipcode | city |		
			จริง ๆ		
			👉 zipcode → บอก city ได้		
			ดังนั้น ต้องแยก Zipcode Table
