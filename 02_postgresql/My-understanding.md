### JOIN Concept			
	1. INNER JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK
	2. LEFT JOIN : 	มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่ง PK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null
	3. RIGHT JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่ง FK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null
	4. FULL OUTER JOIN : มีข้อมูลที่เหมือนกันระหว่าง PK กับ FK และเอาข้อมูลทั้งหมดฝั่งทั้งฝั่ง PK และ FK ถ้าข้อมูลที่เหมือนกัน ไม่มีข้อมูลนั้นจะแสดงค่า Null

### SQL Normalization					
	1. 1NF				
		ทำให้ทุกคอลัมน์มีค่าเดียว แยกย่อยไม่ได้	
	2. 2NF
		- สืบเนื่องจาก 1NF คือ หากต้องมร Row ที่ต้องมีหลายค่าใน Column และ Row เดียวกัน ควรแยกตาราง
		- ไม่มี PK ที่ซ้ำกันใน Row
		- หากออกแบบตารางที่ต้องมี PK ที่ซ้ำกันใน Row ควรที่จะแยกตารางออกมา		
	3. 3NF
		- คอลัมน์ที่ไม่ใช่ key ต้องขึ้นกับ Primary Key เท่านั้น
		- ห้าม column พึ่ง column			
		- Column ที่ขึ้นกับ column อื่น (ที่ไม่ใช่ PK)			
		เช่น			
			| customer_id | zipcode | city |		
			จริง ๆ		
			zipcode → บอก city ได้		
			ดังนั้น ต้องแยก Zipcode Table

### PostgreSQL Database Features
	1. Tables
		- ตารางคือโครงสร้างหลักในการจัดเก็บข้อมูล
		- เก็บข้อมูลในรูปแบบ Row, Column
		- ตารางเป็นเสมือน "ภาชนะ" หลักที่ทำหน้าที่เก็บข้อมูลจริงทั้งหมดในฐานข้อมูลก่อนที่จะมีการสร้างวัตถุอื่น ๆ เช่น Index หรือ View มาเสริมการทำงาน
	2. Indexes
		- เป็นโครงสร้างข้อมูลที่สร้างแยกออกมาเพื่อช่วยให้การค้นหาข้อมูลทำได้รวดเร็วยิ่งขึ้นในคอลัมน์ที่ระบุ
	3. Views
		- View คือ "ตารางเสมือน" (Virtual Table): มันทำหน้าที่เสมือนเป็นตารางหนึ่ง
		- สิ่งที่ View เก็บจริงๆ คือคำสั่ง SQL Query ที่เราตั้งชื่อกำกับไว้
	4. Triggers
